# Yusi 开发哲学与产品理念

---

## 核心理念总览

Yusi 的开发与演进由两条互为表里的理念驱动：

| 维度 | 理念 | 核心内涵 |
|:---|:---|:---|
| **开发哲学** | 以 AI 为中心 | 将平台视为一个 agent 智能体，基于此去设计、开发、演进产品 |
| **产品理念** | 认识自我，遇见同频 | 通过 AI 深度理解每个人的叙事与选择，让真正同频的灵魂相遇 |

二者一体两面：**以 AI 为中心**决定了平台"如何被构建"；**认识自我，遇见同频**决定了平台"为何而存在"。

---

## 1. 开发哲学：以 AI 为中心

### 1.1 平台即 Agent

传统应用将 AI 视为一个被调用的"功能模块"——按需调用，用完即弃。Yusi 从根本上逆转这一关系：

> **Yusi 本身就是一个 agent 智能体。** 用户不是在"使用一个带 AI 功能的应用"，而是在与一个有记忆、能理解、会成长的智能体持续互动。

这意味着：

- **AI 是平台的主体，而非附庸。** 代码架构围绕 AI 的"感知-记忆-推理-行动"循环设计，而非围绕传统的 CRUD 流程。
- **平台会"成长"。** 随着用户持续记录与互动，AI 对用户的理解不断加深——这不是功能叠加，而是智能体的认知演进。
- **AI 驱动进化。** 平台的功能优化与迭代方向，由 AI 对用户需求的理解和对产品数据的洞察来驱动，而非仅仅依赖产品经理的主观判断。

### 1.2 记忆系统：智能体的"自我"

人类的自我认知建立在记忆之上。同样，一个 agent 智能体的"自我"，由其记忆系统定义。

Yusi 的分层记忆架构（短期 → 中期 → 长期），模拟的就是人类的记忆机制：

```
用户交互 → 短期记忆（会话上下文，50 条消息窗口）
                    ↓ AI 总结压缩
              中期记忆（近期重要事件摘要，30 天保留）
                    ↓ 实体抽取 + 关系构建
              长期记忆（人生图谱，永久保留）
```

每一层记忆不是孤立的数据存储，而是 agent 理解用户、回答复杂问题、做出匹配推荐的基础。**记忆不是功能，记忆是智能体的存在方式。**

### 1.3 模型路由：智能体的"思维"

不同的任务需要不同的思维方式。Yusi 的模型路由框架让 agent 能够根据场景选择合适的"大脑"：

- **情景分析**：需要细腻的行为解读 → 高情商模型
- **记忆总结**：需要结构化语义压缩 → 高性价比模型
- **匹配推荐**：需要深度共鸣判断 → 高推理模型
- **日常对话**：需要流畅自然交互 → 平衡型模型

这不是简单的负载均衡，而是智能体在不同认知任务间的**思维切换**。

### 1.4 MCP 服务：智能体的"开放接口"

Yusi 通过独立的 MCP（Model Context Protocol）服务层，将自身能力以标准化协议对外开放：

- **日记搜索**：外部 AI 可查询用户的过往叙事
- **记忆搜索**：外部 AI 可跨层级检索记忆碎片
- **网络搜索**：内置实时搜索能力，补充外部信息

MCP Server 是 Yusi 对外的**服务门户**——外部 AI 应用（如 Claude Desktop 或其他 LLM agent）通过标准的 Streamable HTTP / SSE 协议接入，经由 gRPC 与 Java 后端通信，获取用户的记忆与日记数据。

> MCP 不是后端调用的工具集，而是 Yusi 作为 agent 向外部世界**暴露自身能力的开放接口**。它让任何遵循 MCP 协议的 AI 客户端都能"与 Yusi 对话"。

### 1.5 以 AI 为中心的设计原则

| 原则 | 说明 |
|:---|:---|
| **Agent First** | 任何功能设计，首先问"智能体能为用户做什么"，而非"系统应该支持什么操作" |
| **Memory Native** | 所有数据最终服务于记忆——让 agent 更了解用户 |
| **Context Aware** | agent 应始终知道自己在对谁说话、在什么场景下、用户的历史如何 |
| **Adaptive Evolution** | 平台功能随 agent 对用户理解的加深而自适应调整 |
| **Tool Autonomy** | agent 自主选择和使用工具，而非被预设的流程驱动 |

## 2. 产品理念：认识自我，遇见同频

### 2.1 "认识自我"（Know your vibe）

德尔斐神庙的箴言穿越三千年，在 AI 时代获得了新的可能性。

Yusi 不只是一个日记应用或社交平台。它是一面**AI 之镜**——通过记录你在具体情境中的选择和行动，AI 帮助你看见自己未曾觉察的模式、倾向与成长轨迹。

> **认识自我，遇见同频。 (Know your vibe, find your tribe.)**
>
> 真正的自我认知不是知道"我是 INFP"或"我喜欢旅行"，而是在回顾自己一次次具体的选择后，理解"我是如何在世界中行动的"。

**具体实现**：
- **情景叙事**：在精心设计的情境中，呈现真实的自己
- **日记与记忆**：记录生命中的重要时刻与选择，AI 从中抽取模式
- **人生图谱**：可视化你的实体关系、情绪变化与成长轨迹
- **AI 对话**：在持续对话中，AI 像知己一样帮你反思和理解

### 2.2 "遇见同频"

自我认知是内向的旅程，而灵魂共鸣是外向的连接。二者不是先后关系，而是相互促进的循环：

> **越了解自己，越容易找到同频的人；越与同频的人交流，越能认识自己。**

Yusi 的匹配不是基于标签的"左滑右滑"，而是基于对**每个人的行为和记忆的深度理解**：

- AI 分析的不是"你喜欢什么"，而是"你在具体情境中如何思考和行动"
- 匹配的依据不是"你们有什么共同爱好"，而是"你们是否拥有相似的精神频率"
- 推荐的理由不是"95% 匹配度"，而是一封**由 AI 撰写的个性化推荐信**

### 2.3 产品理念的设计含义

| 理念维度 | 设计含义 |
|:---|:---|
| **隐私即信任** | 只有绝对安全的记录空间，用户才愿意袒露真实的自己。AES/GCM 加密、客户端密钥管理是理念的技术表达 |
| **叙事优于标签** | 平台不鼓励用户给自己打标签，而是引导用户叙事——讲出具体情境下的选择与想法 |
| **理解先于连接** | 没有深度理解就谈不上真正的连接。AI 必须先"认识你"，才能帮你"遇见同频" |
| **质量先于数量** | 每周一次精心匹配（而非每天大量推荐），体现的是对连接质量的追求 |
| **匿名即安全** | 匹配后的匿名对话，让用户可以无负担地袒露真实，而不必担心社交后果 |

## 3. 两条理念的交汇

"以 AI 为中心"是"如何做"，"认识自我，遇见同频"是"为什么做"。它们交汇于以下几点：

### 3.1 Agent 即知己

AI 不是工具，而是用户的**数字知己**。它记住你的每一次选择，理解你的每一种情绪，在你需要时给予反馈，在你迷茫时帮你反思。

> 一个真正的知己，不会只记住你的标签。它会记得你在那个雨天的选择，你在那次冲突中的犹豫，你在那个瞬间的勇气。

### 3.2 Agent 即红娘

AI 也不只是知己，它还是一个**通晓人心的红娘**。它了解每一个人，因此知道谁和谁可能合拍，为什么合拍。

> 一个真正的红娘，不会只看条件匹配。她理解每个人的内核，然后说："我觉得你应该认识这个人，因为……"

### 3.3 Agent 即平台

最终，Yusi 这个智能体既不是单纯的知己也不是单纯的红娘——**它就是平台本身**。用户来到 Yusi，就是在与这个智能体持续互动：记录、反思、被理解、与同频的人相遇。

---

## 4. 演进展望

以 AI 为中心的架构意味着：平台的下一步进化方向，应由 agent 对用户需求的持续理解来驱动，而非仅由外部市场判断决定。

- **感知层进化**：agent 从更多维度理解用户（语音、行为模式、情绪节律）
- **记忆层进化**：更精细的记忆衰减曲线、更智能的记忆融合
- **推理层进化**：从"理解单个人"到"理解人与人之间的关系动态"
- **行动层进化**：agent 主动发起有价值的互动（如"你最近似乎……，想聊聊吗？"）
