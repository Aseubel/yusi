# Yusi 项目总结

---

## 一、项目协调与任务分解

Yusi 采用前后端分离架构，后端基于 Spring Boot 3.4.5 + Java 21，前端基于 React 19 + TypeScript + Vite。开发过程中，通过模块化拆分实现并行开发：后端聚焦 API 服务、AI 集成、数据库设计；前端负责 UI 交互、状态管理、国际化和客户端加密。每日站会同步进度，关键节点 (如加密方案、MCP 协议集成) 提前评审，确保方向一致。

---

## 二、克服的困难

**1. 端到端加密设计**：实现"服务端无法解密"的 CUSTOM 模式，需要在客户端完成加密、服务端仅存储密文。通过 PBKDF2 派生密钥、AES-256-GCM 加密、RSA-OAEP 密钥备份机制解决，同时兼容 DEFAULT 模式保证易用性。

**2. GraphRAG 图谱构建**：在无专用图数据库的情况下，基于 MySQL 实现实体抽取、别名归一化消歧、关系权重累积，支持多跳推理查询。

**3. AI 模型治理**：设计三级熔断状态机 (UP/HALF_OPEN/DOWN)，结合 Redis 发布订阅实现配置热更新，支持 ROUND_ROBIN、WEIGHTED_RANDOM、LEAST_LATENCY、FAIL_OVER 等路由策略。

**4. MCP 协议扩展**：以 Go 语言实现独立的 MCP Server，通过 gRPC 与 Java 后端通信，使外部 AI 能查询用户日记与记忆，实现"数字分身"能力。

## 五、商业推广

1. **目标用户**：注重隐私保护的日记爱好者、追求灵魂契合的社交用户、需要 AI 辅助记忆管理的知识工作者
2. **变现路径**：高级功能订阅 (云同步、更多 AI 能力) → 情感咨询增值服务 → B 端 API 输出
3. **差异化**：端到端加密 + GraphRAG 知识图谱 + MCP 数字分身，形成竞争壁垒
