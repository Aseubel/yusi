## 0. 一切的起源-想法
社交类平台，或为小群体(像棋牌游戏开一个房间那样)提供服务

**背景想法**：不论是线上还是线下，人们结交朋友有一个普遍标准是看对方的言行举止，这个人对待某件事的态度如何很大程度上会改变他人对他的看法。像在社交媒体扩列我们根据对方的言谈来判断是否值得交流。往大了说是打标签，但这样往往是不准确的，我们之所以在相处过程中逐渐熟悉对方，就是在生活小事中去了解对方的思想。

在我看来能说出对方是什么人不算了解，知道对方在某件具体事件处境下会怎么行动才是真的了解。而事实上对于他人行为这一客观事实，不同的人对其想法也不同，这是很难统一的，但是现在有AI就不一样了。

**具体功能**：
①小团体（2-8人）我们可以预设几个具体场景，让用户详细说说自己在这种情形会怎么行动，然后由AI来分析这样的行为，对比所有人的回答给出“合拍程度”，给出性格标签等。

②扩列社交。提供日记平台，让用户将自己以前的事写下，具体自己的行为、过程中的想法，就像给别人讲自己的事那样。某种可能可行的方式让AI记住这些内容生成对用户的看法，类似于一个知心朋友逐步了解用户的过程。AI就像一个交友广泛的人物，这样就可以由他来给用户推荐可能合拍的其他用户。

## 1. 🎯 产品愿景与 v1.0 目标

### 1.1 问题 (The Problem)
当前的社交产品（无论交友还是社区）都存在一个根本问题：它们是“浅”的。
匹配和“扩列”依赖于**“静态标签”（如：喜欢旅游、INFP、美食、90后）。这些标签无法反映一个人的真实个性和价值观。用户在“左滑右滑”中快速消耗，却很难建立起真正的精神共鸣**。

### 1.2 我们的愿景 (The Vision)
我们坚信，真正的理解和连接，来自于：
“了解一个人如何思考和如何行动，而不仅仅是他/她是什么。”
Yusi 将是第一个通过AI深刻理解用户**“叙事” (Narratives)** 和**“情景行为” (Situational Behaviors)**，并以此为唯一标准进行连接的平台。

### 1.3 v1.0 (MVP) 核心目标
*   **验证“钩子” (Hook)**: 通过 "情景室" (Epic 1) 证明用户能从AI分析中立即获得价值和乐趣。
*   **验证“核心” (Core)**: 通过 "AI知己日记" (Epic 2) 证明用户愿意为了“灵魂匹配”而持续贡献他们的“私有叙事”（日记）。
*   **建立“信任” (Trust)**: 从 v1.0 开始，就建立一个“保险库”级别的用户隐私安全感。

## 2. 👥 核心用户画像 (Target Personas)
*   **P1: "灵魂探索者" (The Seeker)**: 厌倦了“标签”社交，渴望找到能“懂我”的知己 (Soulmate) 或真正的朋友 (Friend)。
*   **P2: "自我认知者" (The Explorer)**: 使用 Yusi 的首要动机是“照镜子”，通过 AI 反馈更好地理解自己。
*   **P3: "破冰团体" (The Curious Group)**: 现实中的朋友/同事，寻找新奇、有深度的“派对游戏”或“团建工具”。

## 3. 🚀 v1.0 功能范围 (Functional Scope)

v1.0 的开发工作将仅围绕以下两个核心史诗功能 (Epics) 展开。

### Epic 1: “情景室” (The Situation Room)
**PM目标**: (对应 P1, P3) 打造一个高互动性、高分享欲的“破冰”工具，作为吸引用户的“钩子”。

| 编号 | 功能特性 | 需求描述与验收标准 (Acceptance Criteria) | 优先级 | 状态 |
| :--- | :--- | :--- | :--- | :--- |
| **F1.1** | 房间创建/加入 | - 用户可创建“情景室”，获得一个唯一的6位邀请码。<br>- 房间状态：WAITING, IN_PROGRESS, COMPLETED。<br>- 用户可通过邀请码加入WAITING状态的房间。<br>- 房间人数限制为 2-8人。 | P0 | **[已完成]** |
| **F1.2** | 场景选择与开始 | - 房主可从后台配置的场景库中选择一个场景。<br>- 房主点击“开始”后，房间状态变为IN_PROGRESS。<br>- 场景库支持增删改查。<br>- v1.0 场景库至少包含5个场景。 | P0 | **[已完成]** |
| **F1.3** | 叙事提交 | - 参与者在文本框内（限制100字）叙述自己的“行动”和“想法”。<br>- 在IN_PROGRESS状态下，在自己提交前，不能看到其他任何人的答案。<br>- 提交后，UI显示“等待其他成员...” | P0 | **[已完成]** |
| **F1.4** | AI分析报告 (核心) | - 当所有人都提交后，房间状态变为COMPLETED。<br>- 报告仅房间内成员可见。<br>- 报告包含：【个人分析】、【合拍度矩阵】、【合拍度理由】。 | P0 | **[已完成]** |

### Epic 2: “AI知己日记” (The Confidant Journal)
**PM目标**: (对应 P1, P2) 打造一个“养成系”的AI灵魂伴侣。这是产品的核心价值和长期护城河。它必须是绝对私密、绝对懂你的。

| 编号 | 功能特性 | 需求描述与验收标准 (Acceptance Criteria) | 优先级 | 状态 |
| :--- | :--- | :--- | :--- | :--- |
| **F2.1** | 日记保险库 (The Vault) | - 提供一个绝对私密的日记本功能。<br>- 隐私第一。UI强调“仅用于AI分析”。<br>- **日记条目应支持本地加密存储。** | P0 | **[已完成]** |
| **F2.2** | “AI知己”激活与回应 | - 用户首次撰写日记后，“AI知己”被激活。<br>- 异步生成“回应”或“提问”。<br>- 回应必须体现“理解”。 | P0 | **[已完成]** |
| **F2.3** | “AI知己”聊天 | - 提供一个与“AI知己”的专属聊天界面。<br>- **AI必须能调用用户所有的日记内容（向量记忆库）作为上下文。** | P1 | **[已完成]** |
| **F2.4** | 灵魂匹配设置 | - 提供“灵魂匹配”设置页，默认关闭。<br>- 开启时选择交友意图。 | P1 | **[已完成]** |
| **F2.5** | AI推荐信 (核心) | - 系统定期主动推荐1-3名用户。<br>- 仅显示AI撰写的“匿名灵魂推荐信”。 | P1 | **[已完成]** |
| **F2.6** | 建立连接 | - 只有“感兴趣”和“跳过”两个按钮。<br>- 双方互相点击“感兴趣”后，建立连接。<br>- **在匿名聊天中，双方可进行对话。** | P1 | **[已完成]** |

## 5. ⚠️ 风险与假设
*   **信任风险**: 用户是否愿意写日记？ -> 缓解：加密存储。
*   **内容风险**: AI分析伤人？ -> 缓解：Prompt 约束。
*   **冷启动风险**: 需要数据喂养。 -> 缓解：情景室钩子。
